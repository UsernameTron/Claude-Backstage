**ARCHITECTURE.md** — per FR-1.4, full ADR documenting:

Title: "Package Organization Architecture Decision"

**Context:** 31 Claude Code subsystems extracted from source (~1,900 files, 512K LOC) and KB v2.1 need organizing into a buildable monorepo. Packages are type stubs only. Three tiers reflect HOW each package is used: extract (copy/adapt), build (design reference), translate (new builds from patterns).

**Options Evaluated:**

Option A — Flat: All 31 packages in `packages/` directly.
- Discoverability: Poor at 31+ dirs
- Independence: High
- Dependency mgmt: Simple
- Mixed language: Awkward (Python mixed with TS)
- Context injection: ~150 tokens (long flat list)
- KB v2.1 alignment: None

Option B — Tiered (SELECTED): `packages/{extract,build,translate}/`
- Discoverability: Excellent (tier = usage intent)
- Independence: High (tier is invisible to imports via @claude-patterns/ scope)
- Dependency mgmt: Simple (workspace refs ignore tier path)
- Mixed language: Clean (Python packages isolated in translate/)
- Context injection: ~80 tokens (3 tiers, not 31 names)
- KB v2.1 alignment: 1:1 mapping

Option C — Domain: `packages/{permissions,streaming,ui,...}/`
- Discoverability: Medium (domain grouping)
- Independence: Medium (domain boundaries unclear)
- Dependency mgmt: Complex (cross-domain deps)
- Mixed language: Unclear placement
- Context injection: ~120 tokens
- KB v2.1 alignment: Partial

**Decision:** Option B. Wins on discoverability (tier = usage mental model), KB v2.1 alignment (1:1 tier mapping), and context injection efficiency (~80 tokens). The `@claude-patterns/` scope hides tier directory from import paths, so packages reference each other without knowing their tier.

**Consequences:**
- Workspace globs list tiers, not individual packages
- Python packages excluded from Bun workspaces (pip install -e instead)
- translate/ contains both TS and Python — TS packages listed explicitly in workspace globs
- Tier determines HOW to use, not WHAT it does

Include a decision evaluation matrix table (options x criteria) for easy scanning.
